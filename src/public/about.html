Driven by Purpose, Guided by Curiosity

I like to solve tough problems. I want to work on challenging projects that make a difference for others, surrounded by people smarter than me who inspire me.

Software is about people - those who use it and those who develop and maintain it. I believe in listening to understand needs, perspectives, and obstacles, and formulating a solution to meet them.

Our job as developers is to produce real and measurable results for the users and owners of our software, while also leaving a codebase for fellow developers that is clean, maintainable, and scalable. Nothing is more important to the health of a codebase than quality code, but it means nothing if it doesn’t produce real value for the users and owners of our software in real time.  Our challenge, then, is to produce quality code at the speed of business.

Technical Experience

In my current role, I work as part of a distributed team on a large-scale inventory reporting application. I’ve had the opportunity to lead the re-engineering of this application from a SharePoint On-Premise multipage Site with jQuery to a SharePoint Online SPFx application with React. My contributions have included:

  • Review legacy code to determine re-use and customization
  • Design the new application architecture, leaning heavily on separation-of-concerns, abstraction and encapsulation to maximize scalability and maintainability
  • Generate SharePoint Framework application with multiple WebParts
  • Customize project Gulpfile with custom tasks, custom Webpack configuration and custom Babel configuration
  • Develop custom batch scripts for running the application in the developer Workbench and for packaging the solution for production
  • Develop custom developer-friendly Data API for executing full range of CRUD operations against SharePoint Rest and Graph API using SPFx Context and PnPjs, and against external SQL Server database via custom Power Automate Workflow
  • Develop custom developer-friendly Dialog API leveraging React Context and Material UI Alert and Dialog Components
  • Develop library of custom, re-usable input Components based on Material UI and leveraging use-debounce hook
  • Lead internal code review to ensure adherence to best practices and coding standards
  • Utilize GitLab for source/ version control and for project tracking
  • Prepare external documentation, including release notes for distribution, as well as internal documentation such as versioning strategy and technical instructions for contributing code and managing production workflows

Prior to this, I worked on an internal R&D project, developing new features and functionality for a task management system built on the MERN stack and hosted in AWS. My contributions included:

  • Diagnose and correct unexpected behavior by introducing React Context
  • Develop and maintain both development and production environments on AWS with Elastic Beanstalk and EC2
  • Develop and manage automated CI/CD pipeline with GitLab and AWS CodeDeploy

On these and other projects, I’ve had the opportunity to work in an Agile project environment, collaborating with other developers as well as designers, product owners, and other stakeholders, and had the opportunity to use project management tools like Jira, Confluence, Figma, GitLab and BitBucket.

Professional Legacy

This was a people role in a people business.
